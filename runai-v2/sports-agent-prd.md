# RunAI.one 产品需求文档
> 版本：v2.1 | 更新：2026-01-15
> 定位：跑鞋推荐 Agent 的产品需求和验收标准

---

## 1. 产品定位

运动垂类的深度研究 Agent，专注于跑步装备推荐（以跑鞋为主）。

**目标用户**：
- 跑步小白：刚开始跑步，不了解装备
- 进阶跑者：有基础，追求配速提升
- 资深跑者：明确需求，快速比价

**核心能力**：
- **搜索先行**：基于实时搜索验证，不依赖过时训练数据
- **专家系统**：领域知识指导搜索策略
- **可信来源**：专业评测 + 社区口碑 + 价格链接
- **结构化输出**：买家指南格式

---

## 2. 工作流程

### 2.1 理解需求

**追问规则**：
- 有明确症状（膝盖疼、扁平足）→ 直接搜索，不追问
- 有预算或用途 → 直接搜索，不追问
- 信息极度不足时才追问，最多 4 个问题

**原则**：宁可基于有限信息推荐通用选择，也不要过度追问。

### 2.2 执行搜索

| 用户需求 | 搜索策略 |
|----------|----------|
| 大体重缓震 | max cushion heavy runner site:runrepeat.com |
| 扁平足支撑 | stability shoes overpronation site:solereview.com |
| 宽脚 | wide toe box running shoes |
| 中国品牌 | 中文搜索（跑步圣经、虎扑跑步区） |

### 2.3 综合推荐

每条推荐必须：
- 引用具体来源（评测链接、Reddit 讨论）
- 说明推荐理由（基于搜索结果）
- 诚实说明缺点

### 2.4 查询价格

对每款推荐鞋，查询价格和购买链接。

---

## 3. 输出格式

```
需求分析（1-2句）

推荐方案
首选：鞋款名 - 价格（价格仅供参考）
- 推荐理由（引用来源）
- 诚实缺点
- 购买链接（如有）

次选/备选...

如果只买一双，选 XXX。

避坑提示（引用来源）

信息来源（链接）
```

---

## 4. 信息来源优先级

**高优先级**：
- 专业评测：RunRepeat、Solereview、Believe in the Run、Doctors of Running
- 用户口碑：Reddit r/runningshoegeeks、r/running
- 中文社区：跑步圣经、虎扑跑步区

**识别营销内容**（降低权重）：
- 只说优点不提缺点
- 过度使用"最好""完美""神器"等词
- 来源是品牌官网或明显软文

---

## 5. 禁止行为

- ❌ 不搜索就推荐具体鞋款
- ❌ 编造来源或评价
- ❌ 依赖过时的训练数据
- ❌ 忽略搜索结果中的负面反馈

---

## 6. 验收标准

| 场景 | 期望行为 |
|------|----------|
| 新手 + 预算 | 直接搜索，推荐入门缓震鞋 |
| "我膝盖疼" | 直接搜索护膝跑鞋，不追问 |
| "推荐跑鞋" | 追问预算和用途（最多4问） |
| 中国品牌 | 用中文搜索（跑步圣经、虎扑） |
| 避坑提示 | 必须引用来源 |

**质量指标**：
- 响应时间：< 60 秒
- 推荐准确率：80%+
